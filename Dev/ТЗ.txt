# Техническое задание: «CURLING ROYALE» (версия 1.0)

**Тип документа:** Техническое задание для разработки игры под WebGL (Unity)  
**Платформы:** Яндекс.Игры, VK Play  
**Дата:** 2026-04-10  
**Статус:** Утверждено  

---

## 1. Общие требования

### 1.1. Цель разработки
Создать гиперказуальную игру в жанре Battle Royale с управлением «зажатие мыши — разгон», физикой столкновений и системой прогрессии (рейтинг, валюта, скины, челленджи).

### 1.2. Технические требования
| Параметр | Значение |
|----------|----------|
| Движок | Unity 2022.3 LTS или новее |
| Целевая платформа | WebGL |
| Размер билда | ≤ 20 МБ |
| FPS | 60 (стабильно) |
| Язык программирования | C# |
| Сторонние библиотеки | DOTween (анимации), Newtonsoft.Json (сохранения) |
| Поддержка браузеров | Chrome, Edge, Firefox, Safari (последние 2 версии) |

### 1.3. Платформенная интеграция
- **Яндекс.Игры:** YSDK (сохранения, реклама, таблица лидеров)
- **VK Play:** VKBridge (сохранения, реклама, таблица лидеров)

## 2. Игровой процесс (функциональные требования)

### 2.1. Арена
| Требование | Описание |
|------------|----------|
| Форма | Круг радиусом 10 условных единиц |
| Границы | Невидимый коллайдер-бортик с высоким трением (чтобы камни не вылетали мгновенно) |
| Визуал | Тёмно-синий градиентный пол + светящаяся кайма по краям |
| Камера | Следит за игроком, не выходит за границы арены, ортографическая (вид сверху) |

### 2.2. Препятствия (колонны)

**Жёсткие колонны:**
- Количество: 3
- Радиус: 0.8
- Урон при столкновении: 10 HP
- Коэффициент отскока: 0.6
- Визуал: Тёмно-серый камень с трещинами
- Звук: Глухой удар

**Упругие колонны:**
- Количество: 2
- Радиус: 0.8
- Урон при столкновении: 0 HP
- Коэффициент отскока: 1.2 (ускорение)
- Визуал: Светло-голубой лёд / пружинистый материал
- Звук: Звонкий «пинг»

**Расположение:** Симметричное, фиксированное (указывается в префабе сцены)

### 2.3. Камни (игрок и боты)

| Параметр | Значение |
|----------|----------|
| Размер | Радиус 0.5 |
| Максимальное HP | 100 |
| Базовое замедление | velocity \*= 0.995 за кадр |
| Максимальная скорость | 16 ед/сек |
| Минимальная скорость для удара | 3 ед/сек (ниже — коллизия не наносит урон) |

**HP-бар (над каждым камнем):**
- Ширина: 2 ед, высота: 0.2 ед
- Зелёная полоса (текущее HP) — заполнение слева направо
- Красный фон (всегда виден)
- Анимация при получении урона: зелёная полоса дёргается и сдвигается влево (DOTween)

**Скины:**
- Тело камня: основной спрайт (цвет/текстура)
- Кайма: наложенный сверху спрайт (чуть большего размера) для различия игроков

### 2.4. Управление игроком

| Действие | Реакция |
|----------|---------|
| Зажатие ЛКМ (на камне или в любом месте арены) | Появляется линия-указатель направления и кружок силы (растёт от зелёного к красному) |
| Удержание | Зарядка силы (0.3–2.0 секунды) |
| Отпускание | Разгон в выбранном направлении со скоростью `clamp(chargeTime * 8, 3, 16)` |
| Отмена (отпускание вне арены) | Бездействие |

**Визуальная обратная связь:**
- Линия указывает направление
- Кружок вокруг камня растёт и меняет цвет (зелёный → жёлтый → красный)
- Звук нарастающей зарядки

### 2.5. Расчёт урона от столкновения

**Формула:**
```csharp
float angle = Vector2.Angle(victim.velocity, hitDirection);
float damageMultiplier = 1 - (angle / 180f);
int damage = 0;

if (angle >= 170) damage = 40;          // в спину
else if (angle >= 60) damage = 15 + (int)((angle - 60) / 60f * 10); // 15-25
else if (angle >= 30) damage = 5;       // полубоком (минимальный)
else damage = 0;                         // в лоб
```

**Дополнительно:**
- При скорости камня < 3 ед/сек урон не наносится (просто толчок)
- После нанесения урона у жертвы иммунитет 0.4 секунды

### 2.6. Смерть и окончание раунда
- HP ≤ 0 → камень разбивается (анимация осколков, удаление)
- Вылет за пределы арены → мгновенная смерть
- Раунд заканчивается, когда остаётся 1 камень (победитель)

## 3. Боты (AI)

### 3.1. Количество
5 ботов за раунд

### 3.2. Параметры ботов
| Параметр | Значение |
|----------|----------|
| Максимальная зарядка | 1.5 сек (слабее игрока) |
| Задержка перед ударом | 0.2 сек (имитация реакции человека) |
| Точность угла | Разброс ±15° |
| HP | 100 (как у игрока) |

### 3.3. Поведение (FSM)

**Состояние «Выбор цели» (0.5 сек):**
- 60% — ближайший камень (игрок или бот)
- 30% — уйти от опасности (если рядом враг с высокой скоростью)
- 10% — случайная точка (для хаотичности)

**Состояние «Прицеливание» (0.3–0.7 сек):**
- Бот пытается зайти в спину цели (смещает направление на 30–60°)

**Состояние «Зарядка» (0.5–1.5 сек случайно):**
- Визуально: вокруг бота появляется кружок зарядки (как у игрока)

**Состояние «Удар»:**
- Разгон в выбранном направлении

### 3.4. Анти-имба
- Боты никогда не бьют в лоб, если есть возможность ударить сбоку
- Если HP бота < 30, он пытается убежать (выбирает цель «дальний край арены»)

## 4. Система усилений (перед раундом)

### 4.1. Экран выбора
- Появляется после загрузки раунда (до старта движения)
- Показывается 3 случайных усиления
- Игрок выбирает одно (остальные блокируются)
- Таймер выбора: 10 секунд (по умолчанию — первое усиление)

### 4.2. Список усилений (6 штук)

| ID | Название | Эффект | Иконка |
|----|----------|--------|--------|
| 1 | Острый край | Урон в спину 40 → 55 HP | Меч |
| 2 | Каменная кожа | Макс. HP 100 → 130 | Щит |
| 3 | Быстрый старт | Скорость после удара ×1.15 | Молния |
| 4 | Ледяное трение | Замедление 0.995 → 0.998 | Снежинка |
| 5 | Скользкий удар | Отскок от упругой колонны ×1.5 | Пружина |
| 6 | Боевой дух | При HP < 30 урон в спину +20% | Пламя |

### 4.3. Стоимость
- Первое усиление: 50 осколков **ИЛИ** просмотр рекламы
- Смена выбора (кнопка «Другой вариант»): 20 осколков + реклама (не более 3 смен за раунд)

## 7. Челленджи и события

### 7.1. Ежедневные челленджи (4 штуки)
Сброс в 00:00 по UTC+3

| Челлендж | Условие | Награда |
|----------|---------|---------|
| Снайпер | Нанести 200 HP урона ударами в спину | 100 осколков |
| Живучий | Остаться с HP < 20 и победить | 150 осколков |
| Экспериментатор | Использовать 3 разных усиления за день | 200 осколков |
| Разрушитель | Врезаться в жёсткие колонны 5 раз за раунд и выжить | 100 осколков |

**Прогресс отслеживается в реальном времени через события урона и окончания раунда.**

### 7.2. Ежедневное событие
Одно активное событие в день (меняется автоматически)

| День | Событие | Эффект |
|------|---------|--------|
| Пн | Ледяной шторм | Все колонны становятся упругими |
| Вт | Тяжёлые камни | Урон от столкновений +20% |
| Ср | Ускоренный режим | Зарядка удара в 2 раза быстрее |
| Чт | Хрупкие камни | Все камни имеют 70 HP (вместо 100) |
| Пт | Битва титанов | Все камни имеют 200 HP |
| Сб | Без тормозов | Замедление отключено (velocity не уменьшается) |
| Вс | Бонусный день | Награда за челленджи ×2 |

### 7.3. Еженедельный челлендж
- Сброс по понедельникам
- Один сложный челлендж
- **Пример:** «Король рикошета» — убить врага, используя отскок от упругой колонны
- **Награда:** Эксклюзивный скин (недоступен за осколки) + 500 осколков

## 11. Технические требования к коду

### 11.1. Архитектура
- **MVC-ish:** Модели (данные) → Контроллеры (логика) → View (UI/визуал)
- **Событийная модель:** C# events / UnityEvents для слабой связанности
- **Singleton только для менеджеров** (не более 5: GameManager, UIManager, SaveManager, AdManager, AudioManager)

### 11.2. Производительность
- Не использовать `GameObject.Find()` в Update
- Object pooling для эффектов (осколки, частицы)
- Отключить коллайдеры у мёртвых камней

### 11.3. Физика (собственная реализация)
**Не использовать Rigidbody2D для движения.**  
**Альтернатива:**
```csharp
public class CustomPhysicsBody : MonoBehaviour {
    public Vector2 velocity;
    public float mass = 1f;
    public void ApplyForce(Vector2 force) { velocity += force / mass; }
    void Update() { transform.Translate(velocity * Time.deltaTime); }
}
```

**Коллизии:** `OnTriggerEnter2D` для обнаружения + ручной расчёт отскока.

### 11.4. Детерминизм в WebGL
- Все расчёты урона и отскока — через `float` с округлением до 2 знаков
- Фиксированный `Time.fixedDeltaTime = 0.02` (50 кадров физики в секунду)

## 15. Риски и их митигация

| Риск | Вероятность | Решение |
|------|-------------|---------|
| Физика расходится на WebGL | Средняя | Использовать собственную физику (без Rigidbody) |
| Боты слишком сильные | Высокая | Уменьшить maxChargeTime до 1.2 сек, добавить разброс угла |
| Реклама не загружается | Средняя | Резервный вариант: дать осколки без рекламы (но с таймером 30 сек) |
| Retention низкий (<15%) | Высокая | Добавить ежедневные бонусы и push-уведомления (через API платформы) |
